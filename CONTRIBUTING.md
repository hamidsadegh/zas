# Contributing to ZAS

Thank you for your interest in contributing to ZAS! To ensure consistency and quality, please follow these guidelines:

## Code Style
- Use **PEP8** conventions
- Use meaningful variable and function names
- Write docstrings for new functions, classes, or modules

## Branching and Commits
- Use feature branches: `feature/<feature-name>`
- Use commit messages like: `[feature] Add Device List Pagination`

## Testing
- Use the `tests/` folder under each app
- Run tests before committing: `python manage.py test devices`

## Documentation
- Update `docs/` whenever new features are added
- Include docstrings in `models.py`, `views.py`, and `serializers.py`

## Pull Requests
- Make sure your branch is up-to-date with `main`
- Provide a summary of changes in the PR description
